# CelsianJS Production-Readiness Audit & Fix Plan — 2026-06-07

Combined results of `/smoke-audit` (public surfaces), `/product-review` (strategy),
and a deep source production-readiness review. Local gates were all green at audit
time (build, typecheck, biome lint, **1364 tests pass**, `verify:publish` packs +
installs 20 packages into a clean consumer). The engine is genuinely solid; the
issues below are localized defects + release-hygiene + go-to-market.

## Verdict

- **Software quality:** production-grade engine, strong security defenses, well tested.
- **Blocking for "production-ready" claim:** 4 reproduced CRITICAL code bugs + 1 CRITICAL
  release bug (uninstallable scaffold). All localized and fixable.
- **Go-to-market:** lagging (positioning/proof/docs) — tracked but not release-blocking.

---

## CRITICAL (release-blocking)

1. **Prototype pollution in JSON body parser** — `packages/core/src/body-parser.ts:74`.
   `JSON.parse(text)` is stored to `parsedBody` with no scrub, while query/cookie/param
   parsers all strip `__proto__`/`constructor`/`prototype`. Reproduced: `{"__proto__":{"polluted":"x"}}`
   pollutes `Object.prototype` via a naive user merge. **Fix:** scrub dangerous keys recursively after parse.
2. **Cron `*/0` (and `NaN`/out-of-range) infinite loop DoS** — `packages/core/src/cron.ts:45-54`.
   `for (i=start; i<=max; i+=step)` with `step=0` never terminates; `parseInt` NaN and
   out-of-range values silently accepted. Reproduced: `parseCronExpression('*/0 * * * *')` hangs forever.
   **Fix:** validate `step>0`, reject NaN, bounds-check every value → throw `CelsianError`.
3. **Response cache replays `Set-Cookie`/per-user headers across users** — `packages/cache/src/response-cache.ts:112,128-143`.
   Reproduced: user1's `set-cookie: session=...` served to user2 on cache HIT. **Fix:** strip
   per-user/hop-by-hop headers before caching; allowlist cacheable headers.
4. **Rate limiter bypassable when no proxy header present** — `packages/rate-limit/src/index.ts:80`.
   Fallback key `anonymous-${Date.now()}` is unique per request → unlimited throughput for
   direct (no-proxy) traffic with `trustProxy:true`. Reproduced: 10 reqs, 0 blocked. **Fix:** fail closed (shared bucket or reject).
5. **`create-celsian` scaffolds an uninstallable project** — `packages/create-celsian/src/templates/*.ts`.
   Templates pin `@celsian/core@^0.3.18`, `@celsian/jwt@^0.3.18`, `celsian@^0.3.18` etc. — versions
   never published (core went 0.3.16→0.4.0). `npm install` → ETARGET. **Fix:** pin to actual published
   versions and make version-map drift impossible going forward.

## HIGH

6. **Redis queue non-atomic `pop()` + no in-flight reclaim** — `packages/queue-redis/src/index.ts:73-79`.
   `rpop` then separate `hset`; crash loses message; inflight entries past `visibilityTimeout` never reclaimed.
   **Fix:** atomic LMOVE/Lua into processing list + reaper.
7. **ioredis clients have no `'error'` listener** — `queue-redis/src/index.ts:33`, `ws-redis/src/index.ts:65-66`.
   Unhandled `error` event can crash the process during a Redis outage. **Fix:** attach `.on('error')` to every owned client.
8. **`trustProxy` blindly trusts `X-Forwarded-Host`/`-Proto`** — `packages/core/src/app.ts:536-542`.
   Host-header injection / cache poisoning / poisoned reset links. **Fix:** allowlist trusted host(s)/proxies.
9. **Unguarded `decodeURIComponent` crashes the request** — `core/src/serve.ts:122`, `adapter-node/src/index.ts:184`.
   `/%ZZ` throws `URIError` outside try/catch. **Fix:** wrap + 400.
10. **TaskWorker loses errors when queue backend rejects** — `packages/core/src/task.ts:97,142-159`.
    `pop()`/`ack`/`nack` rejections escape fire-and-forget `tick()`; polling silently stops. **Fix:** guard + reschedule.
11. **`adapter-bun`/`adapter-deno` published at `1.0.0`** — falsely signal stable maturity on a 0.x core.
    Almost certainly a publish mistake. **Fix:** align to 0.x cohort + deprecate the erroneous 1.0.0 at release.

## MAJOR

16. Schema detection: TypeBox over-broad, Valibot `_parse` outdated (modern uses `~standard`) — `schema/src/detect.ts:15,19`.
17. Body-parse errors swallowed via `console.error` (parsedBody undefined) — `core/src/body-parser.ts:96`.
18. `formData()` bypasses the byte limit (chunked multipart memory exhaustion) — `core/src/body-parser.ts:86`.
19. In-memory rate-limit store increment is non-atomic read-modify-write — `rate-limit/src/index.ts:43-58` (doc + interface contract).
20. Scaffold's `src/index.ts` never `export`s `app` → `celsian routes` fails out of the box — `create-celsian` templates.
21. `generate rpc` writes `src/rpc/` but scaffold uses `src/routes/rpc.ts` — convention mismatch — `cli`.

## Resolution log (2026-06-07)

All five workstreams landed; an adversarial reviewer pass + full integration gate followed.

- **All 5 CRITICALs fixed** with reproduced-bug regression tests (proto pollution, cron DoS,
  cache cookie leak, rate-limit bypass, create-celsian uninstallable scaffold).
- **All HIGH/MAJOR/MINOR code items fixed**; 2 items found to be non-bugs after investigation
  (rate-limit in-memory increment was already atomic — proven; documented the store contract).
- **Integration gate caught a cross-workstream regression:** B's response-cache *allowlist*
  stripped security headers (nosniff/CSP/HSTS) from cached responses → switched to a *denylist*
  of credential headers (preserves security/CORS/custom headers, still blocks the per-user leak).
- **Reviewer caught an incomplete CRITICAL fix:** the JWT cross-app guard fix bound at
  guard-creation time (defeatable by reordering register/create) → re-fixed to resolve per-app
  config from the request at request time (order-independent); added the reviewer's defeat case
  as a regression test. Also guarded the generated adapter-node decode, the cookie decode, and
  replaced a bare `Error` with `CelsianError`.
- **Build-only type error fixed:** `schema/detect.ts` cast needed an `unknown` bridge — caught by
  `tsc -b` (CI build), which `tsc --noEmit` (typecheck) did not surface.

Final gate: `pnpm build` ✓ · `pnpm typecheck` ✓ · `pnpm lint:ci` ✓ ·
`pnpm test` **1422 passed / 7 skipped** ✓ · `pnpm verify:publish` (20 packages packed,
clean consumer install + generated-app smoke) ✓.
